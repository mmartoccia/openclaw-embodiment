# Spec: LocalMLXTransport (v1.2)

**Location:** `openclaw_embodiment/transport/mlx.py`
**Status:** Implemented

---

## Purpose

A `TransportHal` implementation that runs inference locally via MLX (Apple Silicon) instead of routing to the OpenClaw cloud gateway. Enables fully offline operation with sub-100ms inference latency and zero cloud dependency.

## Problem Solved

All existing transports require network access to the OpenClaw gateway. `LocalMLXTransport` makes the SDK work completely offline -- critical for mobile robots, demos without network, and privacy-sensitive deployments.

---

## API Surface

```python
from openclaw_embodiment.transport.mlx import LocalMLXTransport, ModelSpec

transport = LocalMLXTransport(
    model_id="mlx-community/Qwen3-0.6B-4bit",
    fallback=http_transport,  # Optional: fallback when MLX unavailable
)

if transport.is_available():
    transport.load_model()
    response = transport.send(context_payload)
else:
    print("mlx_lm not installed")

response = await transport.send_async(context_payload)
```

### `ModelSpec`
```python
@dataclass
class ModelSpec:
    model_id: str = "mlx-community/Qwen3-0.6B-4bit"
    quantization: str = "4bit"
    max_tokens: int = 256
    temperature: float = 0.3
```

### `LocalMLXTransport` methods
| Method | Returns | Description |
|--------|---------|-------------|
| `is_available()` | `bool` | Check mlx_lm installed |
| `load_model()` | `None` | Lazy-load weights into unified memory |
| `unload_model()` | `None` | Free unified memory |
| `send(context)` | `AgentResponse` | Run inference |
| `send_async(context)` | `Awaitable[AgentResponse]` | Async inference |
| `get_expected_latency_ms()` | `int` | Returns 5 (in-process) |

## Data Schemas

Input: `ContextPayload` (aliased as `DeviceContext`)
Output: `AgentResponse` from `context.models`

---

## Hybrid Routing Config

```yaml
transport: local_mlx
model: mlx-community/Qwen3-0.6B-4bit
fallback: http
fallback_threshold_ms: 200
```

If `mlx_lm` is not installed, `send()` routes to `fallback` transport automatically.

---

## Integration Points

- **TransportHal:** Implements full interface (send, receive, connect, etc.)
- **EmbodimentSDK:** Register via `registry.register_transport(LocalMLXTransport())`
- **AttachmentTransport:** Can chain as fallback
- **iOS companion profile:** MLX-Swift variant for iPhone 17 Pro

---

## Failure Modes

| Failure | Handling |
|---------|----------|
| mlx_lm not installed | `ImportError` with pip install instructions |
| Model download fails | `RuntimeError` propagated from mlx_lm |
| Inference exception | Logged; re-raised |
| No fallback + unavailable | `ImportError` |
