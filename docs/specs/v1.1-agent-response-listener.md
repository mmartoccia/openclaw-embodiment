# Spec: AgentResponseListener -- Gate 4 (v1.1)

**Location:** `openclaw_embodiment/core/response.py`
**Status:** Implemented (Gate 4 Complete)

---

## Purpose

Closes the bidirectional loop: `device -> agent -> device`. The existing pipeline sends context to the agent but receives nothing back. `AgentResponseListener` wraps OpenClaw's `onAgentEvent` hook and routes agent responses to `AudioOutputHal` and `DisplayHal` in parallel, completing the full embodiment feedback loop.

## Problem Solved

Gate 3 delivered the **send** half: device fires trigger -> SDK posts context -> agent processes. Gate 4 delivers the **receive** half: agent generates response -> SDK routes to speaker and display. Without this, Reachy never actually reacts to what the agent says.

---

## API Surface

```python
from openclaw_embodiment.core.response import (
    AgentResponseListener,
    DeviceResponseRouter,
    ResponseType,
    AgentResponse,
)

# 1. Build router with device HALs
router = DeviceResponseRouter(
    display_hal=my_display,
    audio_output_hal=my_audio,
    actuator_hal=my_actuator,  # Optional
)

# 2. Build listener
listener = AgentResponseListener(router=router)
listener.register()  # Activate

# 3. On agent event (from webhook or subprocess pipe):
response = AgentResponse(
    response_type=ResponseType.TEXT,
    content="Hello, I see you!",
)
listener.on_agent_event(response)

# 4. Async variant:
await listener.on_agent_event_async(response)

# 5. Register post-dispatch callbacks:
listener.add_callback(lambda r: log(r))

# 6. Shutdown:
listener.deregister()
```

### `AgentResponse` dataclass
```python
@dataclass
class AgentResponse:
    response_type: ResponseType
    content: str
    metadata: Dict[str, Any] = field(default_factory=dict)
    timestamp: datetime = field(default_factory=lambda: datetime.now(tz=timezone.utc))
```

### `ResponseType`
```python
class ResponseType(Enum):
    TEXT    = "text"      # Routes to AudioOutputHal (TTS) or DisplayHal (fallback)
    AUDIO   = "audio"     # Routes to AudioOutputHal only
    DISPLAY = "display"   # Routes to DisplayHal only
    ACTION  = "action"    # Routes to ActuatorHal.execute()
    HEARTBEAT = "heartbeat"  # Acknowledged, no HAL dispatch
```

### `DeviceResponseRouter`
| Method | Description |
|--------|-------------|
| `route(response)` | Dispatch to appropriate HAL |

Routing table:
- `TEXT`: AudioOutputHal.speak_agent_response() → DisplayHal.render_agent_response() (fallback)
- `AUDIO`: AudioOutputHal.speak_agent_response()
- `DISPLAY`: DisplayHal.render_agent_response()
- `ACTION`: ActuatorHal.execute(ActuatorCommand)
- `HEARTBEAT`: log only

### `AgentResponseListener`
| Method | Description |
|--------|-------------|
| `register()` | Activate listener |
| `deregister()` | Deactivate |
| `is_active` | Property: bool |
| `add_callback(cb)` | Post-dispatch callback |
| `remove_callback(cb)` | Remove callback |
| `on_agent_event(response)` | Sync dispatch |
| `on_agent_event_async(response)` | Async dispatch via executor |

---

## Integration Points

- **EmbodimentSDK:** `sdk.response_listener` property; auto-built from registered HALs.
- **AudioOutputHal:** Must implement `speak_agent_response(response: AgentResponse) -> None`.
- **DisplayHal:** Must implement `render_agent_response(response: AgentResponse) -> None`.
- **ActuatorHal:** Receives `ACTION` type via `execute(ActuatorCommand)`.
- **OpenClaw runtime:** Webhook or subprocess from `openclaw agent --stream` pipes events.

---

## Failure Modes

| Failure | Handling |
|---------|----------|
| Router exception | Logged; listener continues |
| Callback exception | Logged; remaining callbacks still fire |
| HAL is None | Silent drop (not an error) |
| Unknown ResponseType | Logged as warning; dropped |
| Deregistered | Events logged and dropped |
