# Spec: StatusIndicatorHal (v1.1)

**Location:** `openclaw_embodiment/hal/base.py`
**Status:** Implemented

---

## Purpose

Provides a uniform interface for LED and visual status feedback across all device profiles. Allows the pipeline to communicate device state (processing, idle, alert) to the user via hardware LEDs without coupling to device-specific LED APIs.

## Problem Solved

Each device has different LED hardware (Reachy Mini front LED, Even G2 indicator light, Pi GPIO LED strip). Without a HAL, the pipeline would need device-specific branches. `StatusIndicatorHal` abstracts this into four core operations: set color, blink, pulse pattern, off.

---

## API Surface

```python
from openclaw_embodiment.hal.base import StatusIndicatorHal, StatusPattern

class MyDeviceLED(StatusIndicatorHal):
    def initialize(self) -> None: ...
    def set_color(self, r: int, g: int, b: int) -> None: ...
    def blink(self, interval_ms: int = 500) -> None: ...
    def pulse(self, pattern: str = "heartbeat") -> None: ...
    def off(self) -> None: ...
    def shutdown(self) -> None: ...
    def validate(self) -> bool: ...
    def get_device_info(self) -> dict: ...
```

### `StatusIndicatorHal` ABC
| Method | Signature | Description |
|--------|-----------|-------------|
| `initialize` | `() -> None` | Power on LED hardware; set to idle state |
| `set_color` | `(r, g, b: int) -> None` | Solid color (0-255 per channel) |
| `blink` | `(interval_ms: int = 500) -> None` | Blink at given interval |
| `pulse` | `(pattern: str = "heartbeat") -> None` | Named animation |
| `off` | `() -> None` | Turn off immediately; always safe |
| `shutdown` | `() -> None` | Release hardware; idempotent |

### `StatusPattern`
```python
class StatusPattern(Enum):
    HEARTBEAT = "heartbeat"   # Slow double-pulse
    ALERT = "alert"           # Fast red blink
    PROCESSING = "processing" # Spinning/sweep
    IDLE = "idle"             # Slow breathing blue
```

---

## Data Schemas

No custom data schemas. All methods use primitive types (int, str).

---

## Integration Points

- **EmbodimentPipeline:** Call `indicator.pulse("processing")` on trigger, `indicator.off()` after actuation.
- **AudioTriggerDetector:** `indicator.pulse("alert")` while in DETECTING state.
- **TriggerDetector:** `indicator.set_color(0, 0, 255)` during FIXATION.
- **AgentResponseListener:** `indicator.pulse("processing")` while agent response is pending.
- **Simulator:** `SimulatedStatusIndicator` records state for CI tests.

---

## Failure Modes

| Failure | Handling |
|---------|----------|
| Invalid RGB channel (>255, <0) | `ValueError` raised |
| Invalid pattern name | `ValueError` raised |
| Hardware unavailable | Implementation raises; catch at pipeline level |
| `off()` when already off | No-op; always safe |
