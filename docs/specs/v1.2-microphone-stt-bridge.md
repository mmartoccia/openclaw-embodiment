# Spec: MicrophoneHal STT Bridge (v1.2)

**Location:** `openclaw_embodiment/hal/base.py` (MicrophoneHal ABC) + `transport/stt_bridge.py`
**Status:** Implemented

---

## Purpose

Adds `transcribe()` as a first-class method on `MicrophoneHal`, eliminating the need for each device profile to ship its own speech stack. All transcription delegates to the OpenClaw STT runtime via `openclaw stt transcribe`, which routes to Whisper, Deepgram, or any other configured provider.

## Problem Solved

Before this, `MicrophoneHal` captured audio but had no transcription path. Developers would need to integrate a per-device ASR stack (local Whisper, cloud API, platform ASR) -- massively increasing per-device complexity. The STT bridge centralizes this in the OpenClaw runtime.

---

## API Surface

### Abstract method on `MicrophoneHal`:

```python
class MicrophoneHal(HALBase, ABC):
    @abstractmethod
    def transcribe(self, audio: AudioChunk, language: str = "en") -> str:
        """Transcribe audio via OpenClaw STT bridge.
        Returns: transcribed text string."""
        ...

    @abstractmethod
    def transcribe_stream(self, stream: Iterator[AudioChunk]) -> Iterator[str]:
        """Streaming transcription -- yields partial transcripts."""
        ...
```

### `OpenClawSTTBridge` (transport/stt_bridge.py):

```python
bridge = OpenClawSTTBridge(
    provider=STTProvider.OPENCLAW,  # or MOCK for tests
    timeout_s=30,
    openclaw_bin="openclaw",
)
text: str = bridge.transcribe(audio_chunk, language="en")
text: str = await bridge.transcribe_async(audio_chunk, language="en")
```

### `STTProvider`
```python
class STTProvider(Enum):
    OPENCLAW = "openclaw"        # Default: openclaw CLI
    WHISPER_LOCAL = "whisper_local"  # Future: local Whisper
    MOCK = "mock"                # Testing
```

## Data Schemas

Input: `AudioChunk` from `hal.base` (same as everywhere else).
Output: `str` (transcript text, may be empty if no speech detected).

---

## Subprocess Protocol

```bash
openclaw stt transcribe <path_to_wav> [--language <lang>]
```

1. Write `AudioChunk.data` to temp WAV file (16-bit PCM).
2. Invoke `openclaw stt transcribe <path>`.
3. Capture stdout as transcript.
4. Remove temp file unconditionally.

---

## Integration Points

- **MicrophoneHal:** `transcribe()` delegates to `OpenClawSTTBridge`.
- **Pipeline:** After audio capture, call `mic.transcribe(chunk)` to get text for agent context.
- **SimulatedMicrophone:** Returns deterministic mock transcript for CI.
- **Reachy2MicrophoneHAL:** Implements via `OpenClawSTTBridge`.

---

## Failure Modes

| Failure | Handling |
|---------|----------|
| `openclaw` not on PATH | `STTError` raised |
| Subprocess timeout | `STTTimeoutError` raised |
| Empty transcript | Returns `""` (valid -- no speech) |
| Non-zero exit code | `STTError` with stderr message |
| Temp file write fails | `OSError` propagated |

## Test Cases

1. `OpenClawSTTBridge(provider=MOCK).transcribe()` returns deterministic string
2. `SimulatedMicrophone.transcribe()` returns mock transcript
3. `transcribe()` with missing binary raises `STTError`
4. `transcribe_async()` completes without blocking event loop
5. Temp file is always cleaned up (success and failure)
6. `transcribe_stream()` yields at least one partial per chunk
