# Spec: AttachmentTransport (v1.2)

**Location:** `openclaw_embodiment/transport/attachment.py`
**Status:** Implemented

---

## Purpose

A `TransportHal` implementation that attaches raw camera frames and audio clips directly to OpenClaw session turns via `openclaw sessions spawn --attachment`. The agent receives the raw frame as base64-encoded data rather than just a compressed embedding, enabling richer multi-modal reasoning.

## Problem Solved

The standard BLE transport sends compressed context payloads with a 25KB size limit. High-value frames get aggressively JPEG-compressed, losing detail the agent could use. `AttachmentTransport` bypasses this limit by POSTing frames directly to the OpenClaw session system.

## API Surface

```python
from openclaw_embodiment.transport.attachment import (
    AttachmentTransport,
    AttachmentConfig,
    AttachmentTransportError,
)
from openclaw_embodiment.context.models import ContextPayload

config = AttachmentConfig(
    session_id="my-agent-session",
    openclaw_bin="openclaw",
    timeout_s=10.0,
    include_audio=True,
    max_frame_bytes=512 * 1024,
    fallback_transport=ble_transport,
)

transport = AttachmentTransport(config=config)
transport.initialize({"session_id": "session-abc"})

# Low-level: accepts JSON-encoded ContextPayload bytes
result: SendResult = transport.send(json_bytes)

# High-level: accepts ContextPayload directly
result: SendResult = transport.send_context(context_payload)
```

### `AttachmentConfig`
| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `session_id` | `str` | `""` | OpenClaw session identifier |
| `openclaw_bin` | `str` | `"openclaw"` | Path to CLI binary |
| `timeout_s` | `float` | `10.0` | Subprocess timeout |
| `include_audio` | `bool` | `True` | Attach audio clips |
| `max_frame_bytes` | `int` | `524288` | JPEG compress if over this |
| `fallback_transport` | `Optional[TransportHal]` | `None` | On failure, route here |

### `AttachmentTransport` methods
| Method | Returns | Description |
|--------|---------|-------------|
| `send(payload)` | `SendResult` | Parse + attach JSON payload |
| `send_context(context)` | `SendResult` | High-level ContextPayload attach |
| `receive()` | `None` | Not supported (responses via AgentResponseListener) |
| `validate()` | `bool` | Check openclaw binary available |
| `get_expected_latency_ms()` | `int` | Returns 100 (subprocess + LAN) |
| `get_measured_latency_ms()` | `Optional[int]` | Rolling average of last 10 sends |

---

## Subprocess Protocol

```bash
openclaw sessions spawn \
  --attachment /tmp/oc_attach_<hash>.json \
  [--session <session_id>]
```

Attachment JSON format:
```json
{
  "event_id": "evt-1234",
  "device_id": "reachy-mini",
  "timestamp_epoch": 1700000000,
  "has_frame": true,
  "has_audio": false,
  "frame_bytes_b64": "<base64 JPEG>",
  "audio_bytes_b64": ""
}
```

---

## Integration Points

- **EmbodimentSDK:** Register alongside BLE transport as higher-bandwidth option.
- **AgentResponseListener:** Response comes back via event system, not `receive()`.
- **ContextPayload:** `send_context()` converts frame and audio to base64.
- **Fallback:** If subprocess fails, routes to `fallback_transport`.

---

## Failure Modes

| Failure | Handling |
|---------|----------|
| `openclaw` not found | `AttachmentTransportError` raised or fallback |
| Subprocess timeout | `AttachmentTransportError` raised or fallback |
| Non-zero exit | `AttachmentTransportError` raised or fallback |
| Temp file cleanup fails | Logged at DEBUG; not fatal |
| Invalid JSON payload | Treated as raw JPEG frame |
