# Spec: AudioTriggerDetector (v1.1)

**Location:** `openclaw_embodiment/triggers/audio_trigger.py`
**Status:** Implemented

---

## Purpose

Adds sound-energy-based triggering alongside the existing visual (IMU-based) `TriggerDetector`. Enables devices with microphones to initiate context capture when they hear a sound above threshold, without requiring a camera or user gesture. Primary target: Reachy Mini (orient head toward sound source, then visual capture).

## Problem Solved

The existing `TriggerDetector` relies entirely on IMU data (saccade + fixation pattern). Devices like the Reachy Mini can also respond to environmental audio cues -- someone calling out, a door slam, a keyword. `AudioTriggerDetector` handles this path without coupling to the visual pipeline.

## API Surface

```python
from openclaw_embodiment.triggers.audio_trigger import (
    AudioTriggerDetector,
    AudioTriggerConfig,
    AudioTriggerState,
)
from openclaw_embodiment.hal.base import AudioChunk

# Config
config = AudioTriggerConfig(
    threshold_rms=800.0,
    min_duration_ms=300,
    poll_duration_ms=250,
    cooldown_ms=2000,
    device="hw:0,0",
    sample_rate=48000,
    channels=2,
    format="S16_LE",
)

# Usage
def on_trigger(chunk: AudioChunk) -> None:
    print("Voice detected!")

detector = AudioTriggerDetector(on_trigger=on_trigger, config=config)
detector.start()
# ... later ...
detector.stop()

# State access
current_state: AudioTriggerState = detector.state
```

### Classes

#### `AudioTriggerConfig`
| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `threshold_rms` | `float` | `800.0` | RMS energy threshold (0-32767 S16_LE range) |
| `min_duration_ms` | `int` | `300` | Sustained duration above threshold to fire |
| `poll_duration_ms` | `int` | `250` | Burst capture length per poll cycle |
| `cooldown_ms` | `int` | `2000` | Post-trigger cooldown before re-arming |
| `device` | `str` | `"hw:0,0"` | ALSA device string |
| `sample_rate` | `int` | `48000` | Sample rate in Hz |
| `channels` | `int` | `2` | Channel count |
| `format` | `str` | `"S16_LE"` | ALSA sample format |

#### `AudioTriggerState`
```python
class AudioTriggerState(Enum):
    IDLE = "IDLE"
    DETECTING = "DETECTING"
    TRIGGERED = "TRIGGERED"
    COOLDOWN = "COOLDOWN"
```

#### `AudioTriggerDetector`
| Method | Returns | Description |
|--------|---------|-------------|
| `__init__(on_trigger, config)` | -- | Construct with callback and optional config |
| `start()` | `None` | Begin polling in background thread |
| `stop()` | `None` | Stop polling; join thread (3s timeout) |
| `state` | `AudioTriggerState` | Thread-safe state property |

## Integration Points

- **MicrophoneHal:** Can share `get_buffer()` data with `AudioTriggerDetector`.
- **TriggerArbiter:** `AudioTriggerDetector.on_trigger` should be wired into `TriggerArbiter.push()` for multi-modal fusion.
- **Reachy ActuatorHal:** On trigger, the pipeline can orient the head toward DoA before camera capture.
- **StatusIndicatorHal:** LED set to `pulse("alert")` while in DETECTING/TRIGGERED states.

---

## Failure Modes

| Failure | Handling |
|---------|----------|
| `arecord` not installed | `subprocess.CalledProcessError` logged; thread retries after 500ms |
| No audio device | Caught generically; thread retries |
| Callback exception | Logged; detector continues |
| Thread dies | `start()` can be called again |

## Test Cases

1. `AudioTriggerDetector` instantiates without error
2. State machine transitions: IDLE -> DETECTING -> COOLDOWN
3. `_compute_rms` returns 0 for silence, > 0 for signal
4. `stop()` is idempotent (safe to call multiple times)
5. Config defaults are reasonable (RMS > 0, cooldown > 0)
